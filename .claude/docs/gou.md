# 知识库目录下的复选框逻辑优化

1. 复选框状态彻底和文档是否同步脱钩
   1. 也就是默认所有文档无论处于什么状态都是不打勾的
   2. 文档当前处于什么状态只由后面的标识来显示

2. 打勾只为了做批量处理
   1. 当点击开始同步的时候，强行全部已经勾选的文档都进行同步
   2. 新加一个批量删除按钮，将所有打勾的文档都从本地删除，让这些文档处于未同步状态

3. 复选框三态变化
   1. 打勾的复选框所有子文档一定都被打勾了，当父文档被打勾了，子文档就都被打勾
   2. 空的复选框子文档一定都为空，当父文档切换到空了，子文档全部切换到空
   3. 方块填充的复选框，子文档一定都不为勾或者空，当子文档的复选框变化的时候，父复选框根据所有子文档复选框变化的结果决定自己状态
