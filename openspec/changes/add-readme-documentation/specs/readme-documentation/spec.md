## ADDED Requirements

### Requirement: README 文档必须包含项目介绍

README.md 必须包含项目名称（飞猫助手 / FlyCat）和核心功能描述。

#### Scenario: 用户查看项目介绍
- **WHEN** 用户打开 README.md
- **THEN** 用户能看到项目名称"飞猫助手（FlyCat）"和核心功能描述

### Requirement: README 文档必须包含功能特性列表

README.md 必须列出主要功能特性，包括飞书知识库同步、文档本地存储等。

#### Scenario: 用户了解功能特性
- **WHEN** 用户阅读 README.md
- **THEN** 用户能看到项目的功能特性列表

### Requirement: README 文档必须包含技术栈说明

README.md 必须说明项目使用的技术栈（Tauri 2.x、React 19、TypeScript、Ant Design 等）。

#### Scenario: 开发者了解技术栈
- **WHEN** 开发者阅读 README.md
- **THEN** 开发者能了解到项目使用的技术栈

### Requirement: README 文档必须包含安装和使用指南

README.md 必须包含应用的安装步骤和基本使用方法。

#### Scenario: 用户安装应用
- **WHEN** 用户按照 README.md 的安装指南操作
- **THEN** 用户能成功安装并运行应用

### Requirement: README 文档必须包含开发环境配置

README.md 必须包含开发环境搭建步骤，包括依赖安装和开发命令。

#### Scenario: 开发者搭建开发环境
- **WHEN** 开发者按照 README.md 的开发指南操作
- **THEN** 开发者能成功搭建开发环境并启动开发服务器
