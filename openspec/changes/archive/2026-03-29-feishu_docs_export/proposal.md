# feishu_docs_export

## Motivation

获取知识库内文件的网络请求本身很快，但当前项目整体导出速度慢。

**根因分析：**
1. 文档逐个串行处理，每两个文档之间固定 sleep 400ms（commands.rs:2037）
2. 每个文档处理完成后都立即保存 manifest 到磁盘（sync.rs:210），产生大量重复 I/O
3. 没有任何并发处理能力，N 个文档至少需要 N * 400ms 的纯等待时间

**用户观察：** 单个文档获取速度快，但整体同步 100 个文档需要 40 秒以上的纯等待时间，严重拖慢体验。

## Scope

- 移除固定 400ms 延迟
- 引入基于 `std::thread` 的并行文档处理（concurrency = 4）
- manifest 写入改为批量模式：每处理完一个文档仅更新内存状态，每 10 个文档或全部完成时才写磁盘
- 保持现有错误处理、重试、进度上报机制不变

## Out of Scope

- 不引入 tokio/async 运行时（避免大规模重构）
- 不修改前端进度显示逻辑（现有事件机制已足够）
- 不修改 API 客户端层（mcp.rs）

## Success Criteria

- 同步 50 个文档的总时间从 ~20s 降低到 ~5s 量级
- manifest 数据完整性和之前一致
- 进度事件仍然正常触发
- 错误文档仍然正确记录
