# 变更报告：在浏览器打开按钮修复

## 基本信息

- **变更名称**: `fix-open-in-browser-button`
- **流程 Schema**: `spec-driven`
- **归档路径**: `openspec/changes/archive/2026-03-30-fix-open-in-browser-button`

## 变更动机

知识树中的“在浏览器打开”按钮已经对文档和多维表格节点可见，但桌面端实际调用了不适合外部 URL 的打开方式，导致用户点击后可能没有任何可见反馈。

## 变更范围

- 将文档/多维表格的网页跳转改为使用 URL 打开能力。
- 保留原有的成功/失败返回结构，避免影响调用方接口。
- 在首页知识树按钮点击失败时展示错误提示，避免静默失败。

## 设计摘要

- 沿用现有 `@tauri-apps/plugin-opener` 插件能力。
- 复用授权登录流程已经在使用的 `openUrl(...)` 方案，减少平台差异风险。
- 继续由 `openDocumentInBrowser(...)` 统一构造 Feishu URL，前端仅负责展示错误消息。

## 规格影响

- 更新 `tauri-desktop-runtime-and-backend` 能力。
- 新增要求：桌面端必须能从知识树将 Feishu 文档和多维表格链接拉起到系统默认浏览器。
- 新增要求：浏览器拉起失败时，错误信息必须返回给前端以便提示用户。

- [x] 保持结构化成功/失败结果

- [x] 保持浏览器模拟环境下的新标签页回退逻辑
