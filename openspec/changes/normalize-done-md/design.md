## Context

`DONE.md` 由多轮 OpenCat 任务累积而成；早期条目使用 `#序号`、日期粒度不一、部分行在编辑中断裂。`/opencat-work` 已约定推荐格式为 `- [时间] 任务名称 — 变更`，且应保持精简单行。

## Goals / Non-Goals

**Goals:**

- 统一可见的版式问题（断行、未完结句子）。
- 将 2026-04-02 相关完成记录集中到文件前部，与同日起较新记录顺序一致。
- 用 `project-done-log` 规格锁定后续维护方式。

**Non-Goals:**

- 不重写历史条目的技术细节或改写执行者署名。
- 不强制把所有历史日期统一为「分钟级」时间戳（保留原文日期粒度）。

## Decisions

- **最小编辑**：仅修改明显破损行与顺序调整；不批量改写 `#n` 前缀类旧格式（避免引入事实风险）。
- **分隔符**：延续近期条目使用的全角 `—`，与 `/opencat-work` 示例一致。
- **规格落地**：新增独立 capability `project-done-log`，避免与 `readme-documentation` 混淆。

## Risks / Trade-offs

- [误判历史语义] → 顺序调整仅限同日且依据归档文件名时间戳；其余保持不动。
- [单行过长] → 接受较长单行，以符合「一行一条」规范。
